Midterm Instructions

HTML Requirements

  • Create an index.html file and include the html boilerplate for correct document structure
  • Apply semantic html where required such as navigation, lists, sections, header, footer, paragraphs, div, etc.
  • HTML contains all of the content as per the provided mock-up
  • HTML elements have adequate use of attributes (id and classes) and structured in a way that sets you up for styling your css efficiently
  • The site must follow accessibility guidelines using alt text for images, skip links, aria roles, proper HTML syntax and overall showing consideration for accessibility by adding attributes that may assist users.
  • HTML code is validated using https://validator.w3.org/#validate_by_input
  • HTML is properly formatted - neatly spaced and indented

CSS Requirements

  • Create a main.css file and place within a css folder.
  • Use the mobile-first approach when applying styles to your page (default styling is suitable for mobile and properties are adjusted for larger screens when media queries are introduced).
  • The maximum width of the wrapper within each section is 1200px
  • Use google fonts Lato and Karla in your project. Lato will be used for all headings and navigation. Karla will be used for all body text and buttons.
  • Your site should respond to different screen sizes. CSS grid, flexbox, or any combination of the two may be used to create the layout of the page.
  • Add a min-width media query for the screen size of 768px and above and adjust page to match the screenshot
  • Add a min-width media query for the screen size of 1100px and above and adjust page to match the screenshot
  • At least one noticeable animation should be used to enhance the overall user experience of the site using the animation property.
  • All buttons should have hover and focus states with transitions applied.
  • CSS is properly formatted
  • Overall design executed well and closely matches the provided mockup
  • CSS code is validated using https://jigsaw.w3.org/css-validator/#validate_by_input

General Requirements

  • Proper folder structure and naming conventions used
  • Submit proper links for project
